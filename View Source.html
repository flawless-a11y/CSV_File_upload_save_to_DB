Services :  

  1. User management  
  2. Merchant Onboarding with file Parsing  
  3. Catalogue  
  4. Order management 
  5. Merchant Payout 

 

 

User management Details :  
User information is managed here for each user will have a unique user id and field like address, card details, email id, phone number etc.  

  • Sign up POST api to insert signing in and customer details.. With basis validation like one to one mapping with contact details.  
  • Signing GET api to check and authenticate if username and password match of given user , user can login with phone / email / username  
  • A GET Api to fetch user details to be fetched on user id as param or  email id or mobile.   

 

 

Catalogue :  

 

  1. A) Merchant Onbarding with File parsing : 

A csv file parsing utility service to parse a file and update file data in DB of catalogue 

Merchant details will be : Merchant id (unique), name, GST, PAN, Address, Account number, phone number. 


  1. B) catalogue Onboarding with File Parsing:

File parsing service where catalogues offered by Merchant and their quantity. A new product or an older product quantity can be uploaded or updated from here.  
File details Will be :  
Merchant ID  
Product ID (Empty for new product else older product id in case of update of quantity or price) 

Price  
Quantity 

Description  

Category 

 

  1. C) Catalogue information Sharing

Catalogue details sharing service where product info is shared to order management service with  GET api on basis on 
filters like  :  
Category  

Price range 

Merchant id   
Sorting like :  
Price : High to Low  or Low to high  
The above api should be paginated.  

 

  1. D) Post api to decrement quantity after order placed will be used by order management service. 
  2. E) Fetch merchant details via GET api via merchant id. 

 

 

 

Order Management 

User will place order here and  can have multiple products from multiple sellers.  

  • Post api to place order and save order details in DB, use catalogue POST api to decrement quantity or product ordered.  
  • GET api to fetch order history for user by user id / email id / phone number 
  • POST api to Mark the order delivered, once order is delivered  
  • GET api to fetch merchant wise order details for each date sent in param.  

 

 

Merchant Payout 

Merchant earning is transferred from here to merchants account on previous day 

  • Scheduler to start and transfer merchant earnings of previous day 
  • Payout history to merchant on merchant id and date basis along with products id and quantity. 






MERCHANT CATALOGUE

It is a microservice that is a part of shopping website backend . This microservice is responsible for onboarding merchants and providing other microservice with the required data it needs for its operation . 

 
 

Schema : Merchant 

Table: merchant_catalogue 

Merchant{ 

merchantId 

integer($int64) 

merchantName 

string 

gstin 

string 

pan 

string 

address 

string 

accountNumber 

integer($int64) 

phoneNumber 

integer($int64) 

created 

string($date-time) 

modified 

string($date-time) 

} 

Primary Key : (gstin) (user defined) 

It is a composite Id to avoid duplicates while parsing the data from the csv file. 

 
 

Foreign Key : (merchantId) (autogenerated) 

This key is to facilitate uniqueness in internal service communications .  

 
 

Field 

Description 

merchantId (Long)  

 
 

Validation : No Validation required as it is generated internally. 

 
 

This is a unique Id that is generated while each record of the csv file is parsed such that each merchant  has an unique identity for internal data exchange/communications/query instead of gstin . 

merchantName (String) 

 
 

Validation : Data is checked that it should only consist of alphabets and spaces , no other characters or symbols are accepted. Should not be null. 

This is user provided data which will be fetched from the csv file uploaded by the client. 

This need not be unique . 

 
 

gstin(String)  

 
 

Validation : This field should not be null and can be made up of Alphanumeric characters but length of string should strictly be 15. 

This is a user provided input which is fetched from a csv file uploaded by the user . This is the primary key used to differentiate duplicate entries and must be unique. 

 
 

pan(String) 

 
 

Validation : can consist of alphanumeric characters and length should be 10. 

 
 

This is a user-provided input fetched from a csv file and needs to be unique . 

address(String) 

 
 

Validation : No validation checks performed  

This is a user-provided input fetched from a csv file and does not need to be unique . 

accountNumber(Long) 

 
 

Validation : Should only contain digits and the length of the number should be between 9 and 16. 

This is a user-provided input fetched from a csv file and does not need to be unique . 

phoneNumber(Long) 

 
 

Validation : Should only contain digits and length should be only 10. 

This is a user-provided input fetched from a csv file and does not need to be unique . 

created(LocalDate) 

It is auto-generated to track the date of entry of merchants into the table. 

modified(LocalDate) 

It is auto-generated to track the date of modification of merchant details in the table.

PAYOUT SERVICE

It is a microservice that helps in calculating the net amount on each successful order which has to be given to the merchant after taking the commission. It consists of the details of the orders such as orderID, merchantId, total_amount, quantity, commission, net_amount.

Schema: Payout Service

}

Primary Key: orderId

(It is unique Id and is helpful for adding the information into payout service while using the scheduler for every 24 hours.)

Field

Description

orderId(Long)

It is a unique Id generated for every new order placed. 

productId(String)

This is a unique Id that is generated while each record of the CSV file is parsed such that each product has a randomly generated string as an Id.

merchantId(Long)

This is a unique id generated in the merchant catalog when the merchant is onboarded.

quantity(Integer)

It represents the amount of the product added to the order.

total_amount(Double)

It represents the total amount of the order placed. ( multiplication of price and quantity)

commission(Double)

It represents the amount which is taken by the organization. ( It is fixed to be 5%)

Id(Long)

It is a unique Id that is auto-generated and is used for keeping the serialization of the table.

net_amount(Double)

It represents the final amount that is to be given to the respective merchant.

(net_amount = total_amount - commission)

created(LocalDate)

It is auto-generated to track the date of entry of the product into the schema

modified(LocalDate)

It is auto-generated to track the date of modification of the product in the schema
